一、建立地面積水遮罩

因為這張場景的地面是一個帶透視的梯形區域，Codex 不應畫一個普通矩形。

先為事件房定義一個地板多邊形：

const FLOOR_POLYGON = [
  { x: 220, y: 490 },
  { x: 955, y: 490 },
  { x: 1110, y: 760 },
  { x: 105, y: 760 },
];

實際座標要依遊戲 Canvas 尺寸校正。

繪製積水時：

ctx.save();

ctx.beginPath();
ctx.moveTo(FLOOR_POLYGON[0].x, FLOOR_POLYGON[0].y);

for (let i = 1; i < FLOOR_POLYGON.length; i++) {
  ctx.lineTo(FLOOR_POLYGON[i].x, FLOOR_POLYGON[i].y);
}

ctx.closePath();
ctx.clip();

// 在 clip 內畫積水
drawFloodSurface(ctx, floodProgress);

ctx.restore();

這樣水不會蓋到牆壁、床、衣櫃或 UI。

二、水不要從零瞬間鋪滿

事件剛觸發時，先從房間深處、牆角與抽水馬達附近出現小面積積水，再逐漸往玩家方向擴張。

定義：

type FloodStage =
  | "inactive"
  | "seeping"
  | "spreading"
  | "danger"
  | "completed";

狀態時間可以先這樣安排：

階段	時間	效果
inactive	事件前	只有原本的小積水
seeping	0–3 秒	牆角開始出水
spreading	3–9 秒	水面向房門推進
danger	9 秒後	幾乎覆蓋地面並開始傷害
completed	玩家離開後	停止更新

核心狀態：

interface FloodState {
  stage: FloodStage;
  elapsed: number;
  progress: number;
  damageTimer: number;
  rippleTimer: number;
}

更新：

function updateFlood(state: FloodState, dt: number): void {
  if (state.stage === "inactive" || state.stage === "completed") return;

  state.elapsed += dt;

  if (state.elapsed < 3) {
    state.stage = "seeping";
    state.progress = state.elapsed / 3;
  } else if (state.elapsed < 9) {
    state.stage = "spreading";
    state.progress = 0.2 + ((state.elapsed - 3) / 6) * 0.8;
  } else {
    state.stage = "danger";
    state.progress = 1;
  }
}
三、積水面怎麼畫

基礎積水層可以使用半透明深灰藍色，但不要太藍，否則會像游泳池。

function drawFloodSurface(
  ctx: CanvasRenderingContext2D,
  progress: number
): void {
  const canvasWidth = ctx.canvas.width;
  const canvasHeight = ctx.canvas.height;

  const startY = canvasHeight;
  const endY = 470;

  const waterTopY =
    startY - (startY - endY) * easeInOutCubic(progress);

  const gradient = ctx.createLinearGradient(
    0,
    waterTopY,
    0,
    canvasHeight
  );

  gradient.addColorStop(0, "rgba(45, 58, 56, 0.18)");
  gradient.addColorStop(0.45, "rgba(30, 42, 41, 0.30)");
  gradient.addColorStop(1, "rgba(15, 24, 24, 0.46)");

  ctx.fillStyle = gradient;
  ctx.fillRect(0, waterTopY, canvasWidth, canvasHeight - waterTopY);
}

雖然水面是用矩形畫，但外層已經用地板 polygon 做了裁切，所以畫面只會落在地面。

四、增加日光燈反光

你這個房間最明顯的光源是天花板日光燈。

水面出現後，應該在地面中央加入一條模糊、破碎的白色反光。

不要畫完整鏡像，畫長形柔光即可：

function drawFluorescentReflection(
  ctx: CanvasRenderingContext2D,
  progress: number,
  time: number
): void {
  if (progress <= 0.15) return;

  ctx.save();

  ctx.globalAlpha = Math.min(0.28, progress * 0.28);
  ctx.filter = "blur(8px)";

  const flicker = 0.85 + Math.sin(time * 8) * 0.08;

  ctx.fillStyle = `rgba(205, 220, 210, ${0.18 * flicker})`;

  ctx.beginPath();
  ctx.ellipse(
    600,
    610,
    105,
    14,
    0,
    0,
    Math.PI * 2
  );
  ctx.fill();

  ctx.restore();
}

反光位置應依場景燈具位置校正。

五、波紋效果

波紋不要密集生成。每秒約 2–4 個就夠。

每個波紋是逐漸擴大的扁橢圓：

interface WaterRipple {
  x: number;
  y: number;
  age: number;
  duration: number;
  maxRadius: number;
}

更新：

function updateRipples(ripples: WaterRipple[], dt: number): void {
  for (const ripple of ripples) {
    ripple.age += dt;
  }

  for (let i = ripples.length - 1; i >= 0; i--) {
    if (ripples[i].age >= ripples[i].duration) {
      ripples.splice(i, 1);
    }
  }
}

繪製：

function drawRipple(
  ctx: CanvasRenderingContext2D,
  ripple: WaterRipple
): void {
  const t = ripple.age / ripple.duration;
  const radius = ripple.maxRadius * t;
  const alpha = (1 - t) * 0.25;

  ctx.save();

  ctx.strokeStyle = `rgba(190, 205, 196, ${alpha})`;
  ctx.lineWidth = 1.2;

  ctx.beginPath();
  ctx.ellipse(
    ripple.x,
    ripple.y,
    radius,
    radius * 0.28,
    0,
    0,
    Math.PI * 2
  );
  ctx.stroke();

  ctx.restore();
}

波紋位置只能在地板可行走區內產生。

六、牆壁出水

房間進入 seeping 時，牆面底部加入數個滲水源。

不用做動畫貼圖，也可先用細線粒子：

interface WallLeak {
  x: number;
  y: number;
  speed: number;
  length: number;
  alpha: number;
}

畫法：

function drawWallLeak(
  ctx: CanvasRenderingContext2D,
  leak: WallLeak,
  time: number
): void {
  const offset = (time * leak.speed) % leak.length;

  ctx.save();
  ctx.strokeStyle = `rgba(90, 110, 105, ${leak.alpha})`;
  ctx.lineWidth = 2;

  ctx.beginPath();
  ctx.moveTo(leak.x, leak.y + offset);
  ctx.lineTo(leak.x, leak.y + offset + leak.length);
  ctx.stroke();

  ctx.restore();
}

牆面滲水主要應放在：

後牆底部
配電箱下方
管線接頭
抽水馬達附近
牆角接縫

不要從天花板整片落水，否則會變成災難片，不像公寓內部逐漸失控。

八、傷害不要和畫面水位完全綁死

為避免玩家在水剛碰到腳時就開始扣血，傷害應只在 danger 階段啟動。

建議：

事件觸發後 9 秒進入危險階段
每 1.5 秒造成 2 點不可迴避傷害
function updateFloodDamage(
  state: FloodState,
  player: Player,
  dt: number
): void {
  if (state.stage !== "danger") return;
  if (!pointInPolygon(player.position, FLOOR_POLYGON)) return;

  state.damageTimer += dt;

  if (state.damageTimer >= 1.5) {
    state.damageTimer -= 1.5;
    applyDirectDamage(player, 2, "flood");
  }
}

建議這個傷害：

不受防禦減免
不觸發擊退
不播放紅色受擊閃光
只播放窒悶聲、畫面輕微暗角或冷色脈衝

否則會像怪物攻擊。

進入危險階段時：

roomDoor.locked = false;
roomDoor.forcedOpen = true;
roomDoor.interactionEnabled = true;

這符合目前房間事件「玩家要走回門口離開」的既有規則。

floodState.stage = "completed";
floodState.progress = 1;

所有淹水更新必須接入既有唯一遊戲迴圈。

效果由以下部分構成：
1. 依事件房地面多邊形裁切的半透明積水層。
2. 水面由房間深處逐步向房門方向擴張。
3. 日光燈在水面的破碎反射。
4. 隨機橢圓波紋。
5. 牆角、管線與抽水設備附近的滴水／滲水粒子。
6. 玩家在水中移動時生成腳步波紋與小型水花。
7. 危險階段每 1.5 秒造成 2 點不可迴避環境傷害。
8. 危險階段自動打開房門，但不得傳送玩家離場。
9. 玩家離開後停止傷害與動畫升級，保留滿地積水作為事件完成狀態。

淹水狀態：
inactive → seeping → spreading → danger → completed

建議時間：
0–3 秒 seeping
3–9 秒 spreading
9 秒後 danger

積水不得蓋到牆面、家具、角色或 UI。
波紋與滴水粒子必須限制數量並可回收，避免持續配置新物件造成記憶體增長。

核心原則是：**畫面上看起來是在淹水，程式上其實只是地面遮罩逐步擴張。**這會是目前場景最穩定的實作方式。

萬一有玩家真的要等水滿，才進去房間要怎麼呈現？
跟在房間等到水滿

要把這兩種情況分開處理，不能只讓水面遮罩永遠停在地板上。

一、玩家留在房間裡等到「水滿」

不建議真的把水一路模擬到天花板。應在視覺與玩法上分成三個高度階段：

1. 地面淹水

水從腳踝逐步覆蓋地板：

移動速度稍降。
腳步產生波紋。
水面反射逐漸增強。
每隔一段時間造成少量環境傷害。
2. 深水階段

達到設定時間後，房間切換成「深水狀態」。

呈現方式：

水面遮罩上升到角色腰部附近。
角色下半身以水面前景遮罩遮住。
角色移動速度大幅下降。
原本的走路動畫仍可播放，但世界位移縮小。
畫面加入水珠、鏡頭潮濕和低頻悶響。
房間音效逐漸變得像隔著水聽見。
傷害開始提高。

這裡不是按照角色實際高度進行流體碰撞，而是用一條帶透視的「前景水面線」遮住角色下半身。

背景牆面
家具／設備
玩家上半身
水面前景遮罩
玩家下半身被遮住
近景水面反光與波紋
3. 完全淹沒

再繼續留下，進入 submerged 狀態。

此時不要再讓畫面看起來像玩家站在水上，而是切換整體表現：

全畫面套用昏暗、混濁的水下濾鏡。
畫面邊緣有水流扭曲。
環境聲改成悶響和低頻震動。
角色移動非常緩慢。
不再播放一般受擊紅閃。
顯示短暫的「無法呼吸」狀態。
生命按固定頻率快速下降。
房門仍然保持可到達，玩家可自行走出去。
不瞬間把玩家傳送出房間。

例如：

完全淹沒後：
每 1 秒造成 8 點窒息傷害
移動速度剩 45%
攻擊與互動停用
房門位置持續顯示微弱互動輪廓

這樣玩家仍有最後逃生機會，但無法把事件房當成安全區久留。

玩家真的死在裡面

若玩家堅持不離開，就正常死亡。

死亡來源可記錄為：

deathCause: "event_room_drowning"

死亡流程仍使用遊戲既有的輪迴／真正死亡規則，不需要特別救援，也不要由紅怡強制拉走玩家。

死亡畫面可顯示：

水沒有淹出房門。
它只填滿了這個房間。

這也能強化「每個房間是獨立空間」的設定。

二、水已經滿了，玩家離開後又重新進去

這時不應重新播放一次漲水流程，而要保存房間事件狀態。

若狀態為：

fully_flooded

玩家站在走廊打開門時，先看到特殊門口效果。

門外呈現

房門打開後：

門後不是正常房間入口，而是一整面昏暗的水。
水像被看不見的玻璃固定在門框裡。
水不流進走廊。
水面與門框完全垂直。
水中可隱約看到日光燈、管線和漂浮文件。
原本房間的聲音變成沉悶的低頻聲。

這非常符合《無期租寓》：

房間裡已經完全灌滿水，但走廊一滴水都沒有。

玩家進入

玩家跨過門檻後，場景立即切換至完整淹水版本：

直接進入 submerged。
不重新從腳踝水位開始。
套用水下畫面。
出生位置仍在房門內側。
房門保持可返回。
窒息倒數立即開始。
已取得的文件與道具不會重新生成。

玩家可以短暫回去查看，但不能長時間探索。

建議給約 8–12 秒的安全窗口，再開始快速扣血。這能避免玩家因讀取場景或操作延遲，一進門就死亡。

不需要另外生成一整張全新場景，可以沿用原場景，疊加以下效果：

全畫面灰綠色半透明層。
緩慢移動的扭曲紋理。

角色動作速度與實際位移同步降低。

五、事件後續是否一直淹水

建議在玩家拿到 B1 權限後，房間依然維持完全淹水；直到擊敗 B1 王後才改變。

擊敗 積水巡檢員後，該房間狀態更新為：

event.stage = "resolved";

玩家再次進入時：

水已經退去。
地面留下厚重水漬和淤泥。
抽水馬達停止。
文件黏在牆面與天花板。
不再造成傷害。
原本水痕形成的文字已經消失。

例如：

水退得太乾淨了。

這樣事件房便會隨主線產生三個明確版本：

初次調查版本
→ 漲水／完全淹沒版本
→ B1 擊敗後退水版本
給 Codex 的核心要求
事件房的淹水狀態必須持久化。

若玩家停留至水位完全升滿：
1. 房間進入 submerged 狀態。
2. 畫面切換為混濁水下效果。
3. 玩家速度降低，攻擊與一般互動停用。
4. 給予短暫逃生時間，之後持續造成窒息傷害。
5. 房門保持可到達，不得直接傳送玩家離場。
6. 玩家可因溺水正常死亡。

若玩家離開後重新進入 fully_flooded 房間：
1. 走廊門口顯示垂直停留在門框內的水體，水不得流入走廊。
2. 跨過門檻後直接進入 submerged 狀態，不重新播放漲水。
3. 出生點必須位於房門內側，確保玩家可以立即返回。
4. 已取得道具不得重生。
5. 擊敗 B1 王後，事件房切換成 resolved 退水版本。

這樣兩種玩家行為都有完整回應：留在裡面會真的被淹死；事後再闖入，會看到一間被水完整填滿、卻完全不影響走廊的獨立異常空間。
