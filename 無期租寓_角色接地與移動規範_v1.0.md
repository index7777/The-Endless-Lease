CharacterRoot

CharacterRoot 是角色真正的世界座標位置，必須代表角色腳底中心或碰撞體底部的地面接觸基準。

禁止將 CharacterRoot 設在角色圖片中心、胸口或頭部。

VisualRoot

VisualRoot 只負責顯示角色動畫。

若不同動畫素材需要微調，只能調整 VisualRoot 的局部偏移，不得改變 CharacterRoot 或碰撞體的世界位置。

所有局部偏移必須集中記錄在角色動畫設定檔，不得散落在各場景程式中。

CollisionRoot

碰撞體必須跟隨 CharacterRoot，不能跟著每幀圖片輪廓變形。

一般移動期間，碰撞體尺寸應保持穩定。只有蹲下、倒地或特殊動作才可切換碰撞形狀。

4. 角色素材與 Pivot 規則

所有角色動畫必須使用統一的腳底基準。

每個動畫幀必須具有：

相同畫布尺寸。
相同角色縮放比例。
相同腳底 baseline。
相同水平中心基準。
相同 pivot。
相同裁切範圍。

建議 pivot 定義為：

角色兩腳之間的地面接觸中心

不得逐幀自動裁切透明區域。

不得依照每幀角色 bounding box 重新置中。

若來源素材已被不同程度裁切，Codex 必須先建立標準畫布，重新對齊全部幀後才能匯入遊戲。

像素風素材必須使用整數像素位置與 nearest-neighbor；非像素風素材也必須避免角色根節點落在不必要的半像素位置上。

5. 地面偵測

角色不得只依靠固定 Y 座標判斷地面。

Codex 必須建立統一的地面偵測系統，可依引擎使用：

Raycast
Capsule cast
Shape cast
Character controller floor detection
導航地面高度查詢
可行走區域高度圖

地面偵測至少要取得：

is_grounded
ground_position
ground_normal
ground_distance
ground_collider
ground_material

角色落地時，CharacterRoot 必須對齊實際接觸點，而不是只讓腳底圖片看起來接近地面。

地面吸附只可在角色已接近地面且垂直速度合理時生效，不得將正在跳躍或下落的角色強制黏回地面。

6. 地面貼合規則

角色站立或行走時必須維持穩定接地。

建議流程：

先根據輸入計算水平移動。
執行碰撞移動。
取得最新地面接觸結果。
在允許的吸附距離內修正到地面。
更新動畫狀態。
最後更新角色視覺、陰影與音效。

不得先播放動畫後再猜測角色位置。

地面吸附應有最大修正距離與最大速度，避免角色在高低差間瞬移。

參考設定可放入角色設定檔：

{
  "ground_probe_distance": 12,
  "ground_snap_distance": 6,
  "max_ground_snap_speed": 120,
  "grounded_tolerance": 2,
  "max_step_height": 8,
  "max_walkable_slope_degrees": 35
}

實際數值需依遊戲單位與角色尺寸調整，不得直接機械套用。

7. 走路與移動速度同步

角色滑步通常是因為世界移動速度與走路動畫週期不一致。

Codex 必須建立以下關係：

每個完整走路循環的理論移動距離
=
遊戲中的水平速度
×
動畫循環時間

若角色動畫一個循環應前進 stride_distance，動畫循環時間為 cycle_duration，則合理速度為：

move_speed = stride_distance / cycle_duration

動畫播放速度應根據實際移動速度動態調整：

animation_speed_ratio =
actual_horizontal_speed / authored_horizontal_speed

比例必須設定合理上下限，避免動畫過快或過慢。

例如：

0.75 <= animation_speed_ratio <= 1.35

若玩家移動速度明顯超出動畫可接受範圍，應切換到慢走、快走或跑步動畫，而不是無限制加速同一段動畫。

8. 動畫狀態切換

角色動畫至少應有明確狀態：

Idle
WalkStart
WalkLoop
WalkStop
Turn
RunStart
RunLoop
RunStop
Fall
Land
Interact
Attack
Hurt
Dead

不得只使用 Idle 與 Walk 兩個狀態處理所有移動。

開始移動

角色從靜止開始走路時，應使用短暫的起步動畫或合理淡入，不得直接從 idle 任意幀切到 walk loop。

停止移動

角色停止時應：

等待最近的合理接觸幀，或
播放 walk-stop 動畫，或
使用短時間 crossfade 過渡到 idle。

禁止在腳懸空時直接切回 idle。

轉身

改變方向時不得單純水平翻轉正在播放的任意姿勢並立即繼續。

若方向瞬間反轉，應：

短暫減速；
播放轉身動畫；或
至少將動畫切換到合理的轉身／重新起步階段。
9. 腳步接觸與 Foot Lock

走路動畫必須標記左右腳接觸事件：

left_foot_contact
left_foot_release
right_foot_contact
right_foot_release

腳接觸地面期間，該腳在世界座標上不應明顯滑動。

若為 3D 角色，Codex 應優先使用：

Foot IK
Animation Rigging
Ground-normal alignment
Foot locking
Pelvis height compensation

若為 2D 序列幀，Codex 應使用：

固定腳底 baseline
逐幀腳部接觸標記
動畫速度與移動速度同步
必要時對 VisualRoot 做小幅、可設定的局部修正

不得為了鎖腳而移動整個碰撞根節點來回抖動。

10. 角色上下抖動控制

走路動畫中身體自然上下起伏是允許的，但必須只發生在視覺層。

角色世界碰撞根節點不應跟隨頭部或軀幹動畫上下移動。

正確方式：

CharacterRoot：穩定沿地面移動
VisualRoot：呈現走路時的輕微上下起伏

錯誤方式：

CharacterRoot：每幀隨圖片高度上下跳動

Codex 必須檢查角色腳底位置、頭部位置與碰撞體位置，區分自然動畫起伏與錯誤座標抖動。

11. 角色陰影

接觸陰影只能作為輔助深度線索，不能用來掩蓋錯誤位置。

陰影應：

位於角色實際地面接觸點。
不跟著角色身體動畫上下抖動。
角色站立時保持穩定。
跳躍時依離地高度縮小、變淡或擴散。
在斜坡上依地面法線合理變形。
不穿過牆壁或家具。

陰影中心應跟隨 ground_position，而不是角色圖片中心。

12. 台階、斜坡與障礙

角色必須正確處理：

小型門檻
地板高低差
斜坡
樓梯
家具碰撞
狹窄走道
房門附近的碰撞邊界

低於 max_step_height 的高度可以平滑跨越。

高於限制的障礙必須阻擋角色，不得穿透或瞬移到上方。

斜坡上的角色腳底需貼合地面，但角色身體是否隨斜坡旋轉，應依遊戲美術風格決定。2D 側視角色通常保持垂直，只修正腳底高度；3D 角色可透過腳部 IK 適度貼合。

13. Root Motion 原則

Codex 必須明確選擇並統一以下其中一種方案：

方案 A：程式控制移動
遊戲程式控制 CharacterRoot 位移。
動畫只顯示動作。
動畫速度依實際速度同步。
適合大多數 2D 遊戲及需要精準控制的角色。
方案 B：Root Motion
動畫提供角色位移。
程式讀取 root motion 並經過碰撞系統修正。
適合有完整 3D 動作資料的角色。

禁止同時讓程式速度與 root motion 疊加，否則會造成角色加速、滑步或位移重複。

若專案沒有完整可靠的 root motion 資料，預設使用方案 A。

14. 幀率與時間步進

角色移動必須使用時間差計算，不得綁定顯示 FPS。

位置更新使用：

distance = speed × delta_time

物理與動畫需在以下情況測試：

30 FPS
60 FPS
120 FPS
短暫掉幀
暫停後恢復
視窗失焦後恢復

不同幀率下，角色移動距離、落地位置與動畫節奏不得發生顯著差異。

15. Codex 的必要修正流程

每次修正角色移動問題時，Codex 必須依序執行：

確認目前使用的遊戲引擎與角色節點結構。
檢查角色 sprite/model 的 pivot。
檢查動畫幀畫布與腳底 baseline。
檢查碰撞體底部與角色腳底的相對位置。
檢查角色根節點是否在腳底接觸中心。
檢查地面碰撞與 ground probe。
檢查移動速度與動畫週期。
檢查 idle、walk-start、walk-loop、walk-stop 的切換。
檢查陰影是否使用真實地面位置。
建立除錯視覺化。
在實際遊戲場景測試。
輸出修正前後報告。

不得跳過分析而直接修改數值碰運氣。

17. 自動驗證

Codex 必須建立角色移動驗證工具或測試場景，至少檢查：

靜止時腳底與地面距離。
行走時腳底最大漂浮距離。
接觸腳在接觸階段的世界座標滑動量。
角色根節點的垂直抖動。
動畫循環距離與實際位移差異。
停止時是否在合理接觸幀。
左右方向切換是否出現瞬移。
斜坡與台階是否正常。
不同 FPS 下移動結果是否一致。
碰撞體是否穿過地面或家具。
陰影是否位於真實接地點。

建議初始品質限制：

{
  "max_idle_ground_gap_px": 1.0,
  "max_walk_ground_gap_px": 2.0,
  "max_root_vertical_jitter_px": 0.5,
  "max_contact_foot_sliding_px": 2.0,
  "max_speed_cycle_error_ratio": 0.05,
  "max_stop_position_snap_px": 2.0
}

數值可依遊戲解析度調整，但必須記錄原因。

19. 完成標準

只有同時符合以下條件，角色移動才可標記為完成：

腳底與地面穩定接觸。
角色沒有可見漂浮。
碰撞根節點沒有不合理上下抖動。
走路動畫與實際速度一致。
接觸腳沒有明顯滑動。
開始、停止和轉向自然。
idle 與 walk 之間沒有姿勢跳變。
陰影位於真實接地點。
不同房間不需要手動設定角色高度。
不同 FPS 下結果一致。
斜坡、台階與門檻測試通過。
Debug 視覺化與驗證報告已完成。
所有修正均為通用系統，而不是單一場景補丁。

最終判定原則：

角色的世界位置必須由碰撞、地面偵測與移動系統決定；角色動畫只能表現動作，不能偷偷改變角色站立的位置。任何需要逐房間手動修正腳底高度的做法，都代表系統仍未完成。
