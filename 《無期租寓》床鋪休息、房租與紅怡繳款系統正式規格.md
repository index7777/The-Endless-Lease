# 《無期租寓》床鋪休息、房租與紅怡繳款系統正式規格

## 1. 文件目的

本文件定義以下系統的正式行為：

* 遊戲時間與每日循環
* 玩家房間床鋪互動
* 休息與生命恢復
* 次日房租結算
* 欠租狀態
* 紅怡繳款流程
* 休息權限鎖定與恢復
* 存檔資料與狀態一致性
* 系統例外與驗收條件

實作者必須優先整合現有的時間、玩家生命、對話、貨幣、任務與存檔系統。

不得在未確認現有架構前，建立功能重複的第二套時間系統、貨幣系統、對話系統或存檔系統。

---

# 2. 核心設計規則

## 2.1 遊戲時間

現實時間 12 分鐘等於遊戲內完整一天。

```text
現實時間 720 秒 = 遊戲內 24 小時
```

換算：

```text
現實 1 秒 = 遊戲內 2 分鐘
現實 30 秒 = 遊戲內 1 小時
```

每日時間範圍：

```text
00:00 至 24:00
```

時間到達 24:00 時，遊戲進入下一日房租結算。

---

## 2.2 床鋪定位

床鋪是玩家自己房間內的固定互動物件。

床鋪不是一般回血點，而是玩家主動結束當日的功能。

成功休息後：

1. 玩家生命恢復至最大值。
2. 當日剩餘時間立即結束。
3. 遊戲日期增加一天。
4. 遊戲時間設定為次日房租結算時間。
5. 執行每日房租結算。
6. 執行每日世界狀態刷新。
7. 儲存遊戲。

玩家不論在當日何時休息，都會直接前進至次日。

剩餘時間不會轉換成額外資源，也不會降低次日房租。

---

## 2.3 欠租限制

只要玩家存在任何欠租，床鋪完整休息功能立即停用。

欠租判定：

```text
outstandingRentDebt > 0
```

欠租期間，玩家不能：

* 使用床鋪恢復生命。
* 使用床鋪推進至次日。
* 在床鋪介面直接繳款。
* 透過其他 UI 遠端繳款。
* 進行部分繳款。
* 使用自動扣款清償欠款。
* 向紅怡以外的 NPC 繳款。

---

## 2.4 唯一繳款規則

紅怡是遊戲中唯一合法的房租收款入口。

玩家只能：

1. 前往管理處。
2. 與紅怡互動。
3. 選擇繳交欠租。
4. 一次付清全部欠款。

管理處不接受：

* 部分付款
* 分期付款
* 延期付款
* 床邊付款
* 遠端付款
* 其他 NPC 代收
* 自動扣款

玩家持有金額不足以支付全部欠款時，付款必須失敗，且不得扣除任何金錢。

# 3. 系統資料定義

以下名稱為建議語意。實作者可以配合現有專案命名，但行為必須一致。

## 3.1 時間資料

```ts
interface GameTimeState {
  day: number;
  timeOfDayMinutes: number;
  isDayTransitioning: boolean;
}
```

欄位定義：

```text
day
目前居住日數，最小值為 1。

timeOfDayMinutes
當日遊戲時間，以分鐘保存。
有效範圍為 0 至 1439。

isDayTransitioning
是否正在執行跨日、休息或每日結算流程。
流程期間不得再次觸發休息或跨日。
```

---

## 3.2 房租資料

```ts
interface RentState {
  dailyRent: number;
  outstandingRentDebt: number;
  overdueDays: number;
}
```

欄位定義：

```text
dailyRent
目前每日應付房租。

outstandingRentDebt
尚未清償的房租總額。

overdueDays
目前累積欠租天數。
```

不得建立獨立的永久欄位：

```ts
bedLocked: boolean;
sleepPrivilegeSuspended: boolean;
```

床鋪是否可用必須由欠款狀態推導：

```ts
const hasOutstandingRent =
  rentState.outstandingRentDebt > 0;
```

避免出現欠款已結清，但床仍保持鎖定的狀態不同步問題。

---

## 3.3 玩家貨幣資料

應使用專案既有的玩家貨幣資料來源。

概念介面：

```ts
interface WalletState {
  money: number;
}
```

不得因本功能建立第二個玩家金錢欄位。

---

## 3.4 玩家生命資料

應使用專案既有的生命系統。

概念介面：

```ts
interface PlayerHealthState {
  currentHealth: number;
  maxHealth: number;
}
```

成功休息後：

```ts
currentHealth = maxHealth;
```

# 4. 床鋪互動條件

## 4.1 可休息條件

玩家必須同時符合以下條件：

```text
玩家位於自己的房間。
玩家未處於戰鬥狀態。
玩家未處於死亡或倒地狀態。
玩家未處於劇情演出狀態。
玩家未處於場景切換狀態。
目前沒有跨日流程。
目前欠款為 0。
```

建議概念函式：

```ts
type SleepBlockReason =
  | "not_in_player_room"
  | "in_combat"
  | "player_incapacitated"
  | "cutscene_active"
  | "scene_transition"
  | "day_transition_active"
  | "rent_overdue";

interface SleepCheckResult {
  allowed: boolean;
  reason?: SleepBlockReason;
}
```

```ts
function checkCanSleep(): SleepCheckResult {
  if (!worldState.isInPlayerRoom) {
    return {
      allowed: false,
      reason: "not_in_player_room",
    };
  }

  if (combatState.isInCombat) {
    return {
      allowed: false,
      reason: "in_combat",
    };
  }

  if (playerState.isIncapacitated) {
    return {
      allowed: false,
      reason: "player_incapacitated",
    };
  }

  if (storyState.isCutsceneActive) {
    return {
      allowed: false,
      reason: "cutscene_active",
    };
  }

  if (sceneState.isTransitioning) {
    return {
      allowed: false,
      reason: "scene_transition",
    };
  }

  if (gameTimeState.isDayTransitioning) {
    return {
      allowed: false,
      reason: "day_transition_active",
    };
  }

  if (rentState.outstandingRentDebt > 0) {
    return {
      allowed: false,
      reason: "rent_overdue",
    };
  }

  return {
    allowed: true,
  };
}
```

# 5. 正常床鋪互動流程

## 5.1 靠近提示

當玩家靠近床鋪且沒有欠租：

```text
E　休息至次日
```

控制器按鍵應依專案既有輸入系統顯示。

---

## 5.2 休息確認畫面

玩家按下互動鍵後，必須先顯示確認介面，不得立即休息。

建議文字：

```text
完整休息

生命將完全恢復。
今日剩餘時間將直接結束。
遊戲將前進至下一日房租結算。

目前日期：第 {currentDay} 日
目前時間：{currentTime}
下一期房租：${dailyRent}

［休息］
［取消］
```

如果玩家目前持有金額不足以支付下一期房租，可增加警告：

```text
警告：
目前金額不足以支付下一期房租。
結算後將進入欠租狀態，床鋪休息權限將被停用。
```

此警告不得禁止玩家休息。

玩家可以自行選擇承擔次日欠租風險。

---

## 5.3 玩家取消

玩家選擇取消時：

* 關閉確認畫面。
* 不恢復生命。
* 不推進時間。
* 不收取房租。
* 不刷新世界。
* 不修改存檔。

---

## 5.4 玩家確認休息

玩家確認休息後：

1. 將 `isDayTransitioning` 設為 `true`。
2. 鎖定玩家移動與互動。
3. 暫停一般遊戲時間推進。
4. 關閉其他可互動 UI。
5. 播放入睡動畫或畫面淡出。
6. 執行統一跨日流程。
7. 恢復玩家全部生命。
8. 播放每日房租結算畫面。
9. 執行每日刷新。
10. 自動存檔。
11. 播放起床畫面或淡入。
12. 將 `isDayTransitioning` 設為 `false`。
13. 恢復玩家操作。

---

# 6. 統一跨日流程

自然時間跨日與床鋪休息不得各自建立不同的房租結算邏輯。

所有跨日行為必須共用同一個入口。

```ts
type DayAdvanceReason =
  | "natural"
  | "sleep"
  | "scripted";
```

建議概念：

```ts
interface AdvanceDayOptions {
  reason: DayAdvanceReason;
  restoreFullHealth: boolean;
}
```

```ts
async function advanceToNextDay(
  options: AdvanceDayOptions
): Promise<void> {
  if (gameTimeState.isDayTransitioning) {
    return;
  }

  gameTimeState.isDayTransitioning = true;

  lockPlayerControl();
  pauseWorldClock();

  try {
    await playDayTransitionFadeOut(options.reason);

    gameTimeState.day += 1;
    gameTimeState.timeOfDayMinutes = 0;

    const settlement = settleDailyRent();

    refreshDailyWorldState();
    updateDailyNpcState();
    refreshDailyRandomEvents();

    if (options.restoreFullHealth) {
      restorePlayerToFullHealth();
    }

    await showDailySettlementScreen(settlement);

    saveGame();

    await playDayTransitionFadeIn(options.reason);
  } finally {
    resumeWorldClock();
    unlockPlayerControl();

    gameTimeState.isDayTransitioning = false;
  }
}
```

休息呼叫：

```ts
advanceToNextDay({
  reason: "sleep",
  restoreFullHealth: true,
});
```

自然跨日呼叫：

```ts
advanceToNextDay({
  reason: "natural",
  restoreFullHealth: false,
});
```

劇情跨日是否回血由劇情規格決定。

# 7. 每日房租結算

## 7.1 房租產生時機

房租只在日期正式增加時產生一次。

不得在以下時機額外收取房租：

* 開啟床鋪確認畫面時
* 取消休息時
* 與紅怡對話時
* 進入玩家房間時
* 載入存檔時
* 重新進入場景時
* 起床動畫播放時

---

## 7.2 房租支付模式

每日結算時，依目前遊戲既有設計處理當期房租。

本規格建議以下固定模式：

```text
若玩家金額足夠：
立即支付當期完整房租。

若玩家金額不足：
不扣除任何金額。
將完整當期房租加入欠款。
```

概念函式：

```ts
interface RentSettlementResult {
  rentAmount: number;
  paymentSucceeded: boolean;
  amountPaid: number;
  debtAdded: number;
  totalOutstandingDebt: number;
}
```

```ts
function settleDailyRent(): RentSettlementResult {
  const rentAmount = calculateDailyRent();

  if (wallet.money >= rentAmount) {
    wallet.money -= rentAmount;

    return {
      rentAmount,
      paymentSucceeded: true,
      amountPaid: rentAmount,
      debtAdded: 0,
      totalOutstandingDebt:
        rentState.outstandingRentDebt,
    };
  }

  rentState.outstandingRentDebt += rentAmount;
  rentState.overdueDays += 1;

  return {
    rentAmount,
    paymentSucceeded: false,
    amountPaid: 0,
    debtAdded: rentAmount,
    totalOutstandingDebt:
      rentState.outstandingRentDebt,
  };
}
```

如果專案已經定義不同的正常房租支付規則，可以保留既有正常付款方式。

但欠租清償仍必須符合本文件規定：

```text
只能找紅怡。
只能一次付清。
不能部分付款。
```

# 8. 欠租床鋪互動

## 8.1 靠近提示

玩家欠租時，床鋪仍可互動，但只能顯示限制資訊。

建議提示：

```text
E　查看休息限制
```

不得完全隱藏互動提示，否則玩家可能誤認床鋪故障。

---

## 8.2 欠租提示介面

玩家與床互動時顯示：

```text
休息權限已暫停

目前欠繳：${outstandingRentDebt}

請前往管理處，
向紅怡一次繳清全部欠款後，
恢復床鋪休息權限。

［確認］
```

可使用的世界觀補充文字：

```text
居住權仍有效。
完整休息權限已凍結。
```

床鋪介面不得顯示：

* 支付
* 部分支付
* 自動扣款
* 前往繳款
* 使用道具抵扣
* 延後付款

---

## 8.3 欠租時的系統行為

欠租床鋪互動必須保證：

```text
不回復生命。
不改變日期。
不改變遊戲時間。
不刷新每日事件。
不扣除金錢。
不修改欠款。
不啟動跨日流程。
```

---

# 9. 紅怡繳款流程

## 9.1 唯一繳款入口

只有紅怡的對話系統可以呼叫欠租清償函式。

以下系統不得呼叫清償函式：

* 床鋪
* 玩家狀態 UI
* 背包
* 電梯
* 自助機
* 管理處場景本身
* 其他 NPC
* 除錯功能以外的遠端介面

正式遊戲流程中，必須透過紅怡對話選項觸發。

---

## 9.2 紅怡無欠款對話

如果：

```text
outstandingRentDebt === 0
```

紅怡不顯示「繳交欠租」選項。

可以顯示一般對話或主線對話。

---

## 9.3 紅怡有欠款對話

如果：

```text
outstandingRentDebt > 0
```

紅怡對話選項加入：

```text
［繳交欠租］
```

建議對話：

```text
紅怡：
「你的帳還掛著。」
```

選項：

```text
［繳交欠租］
［離開］
```

如同時存在主線對話，應依專案對話優先級處理，但完成主線對話後仍應提供繳租選項。

不得因主線進度永久隱藏繳款入口。

---

# 10. 一次付清規則

## 10.1 付款前檢查

玩家選擇「繳交欠租」時：

```ts
const debt = rentState.outstandingRentDebt;
const money = wallet.money;
```

必須確認：

```text
debt > 0
money >= debt
```

---

## 10.2 金額不足

如果：

```text
wallet.money < outstandingRentDebt
```

付款失敗。

不得扣除任何金額。

建議顯示：

```text
欠款：${outstandingRentDebt}
持有：${wallet.money}
尚缺：${outstandingRentDebt - wallet.money}

紅怡：
「管理處只接受一次結清。
等你湊齊再來。」
```

按鈕：

```text
［離開］
```

系統結果：

```text
玩家金錢不變。
欠款不變。
欠租天數不變。
床鋪仍不可使用。
```

---

## 10.3 金額足夠

如果：

```text
wallet.money >= outstandingRentDebt
```

顯示付款確認：

```text
欠款：${outstandingRentDebt}
持有：${wallet.money}

是否一次付清全部欠款？

［支付 ${outstandingRentDebt}］
［取消］
```

---

## 10.4 玩家取消付款

取消時：

```text
玩家金錢不變。
欠款不變。
床鋪狀態不變。
```

---

## 10.5 玩家確認付款

付款必須視為單一不可分割交易。

建議函式：

```ts
type RentPaymentFailureReason =
  | "no_outstanding_debt"
  | "insufficient_funds"
  | "payment_in_progress";

interface RentPaymentResult {
  success: boolean;
  amountPaid?: number;
  remainingMoney?: number;
  failureReason?: RentPaymentFailureReason;
}
```

```ts
let isRentPaymentInProgress = false;

function payAllOutstandingRentToHongYi():
  RentPaymentResult {
  if (isRentPaymentInProgress) {
    return {
      success: false,
      failureReason: "payment_in_progress",
    };
  }

  const debt = rentState.outstandingRentDebt;

  if (debt <= 0) {
    return {
      success: false,
      failureReason: "no_outstanding_debt",
    };
  }

  if (wallet.money < debt) {
    return {
      success: false,
      failureReason: "insufficient_funds",
    };
  }

  isRentPaymentInProgress = true;

  try {
    wallet.money -= debt;
    rentState.outstandingRentDebt = 0;
    rentState.overdueDays = 0;

    saveGame();

    return {
      success: true,
      amountPaid: debt,
      remainingMoney: wallet.money,
    };
  } finally {
    isRentPaymentInProgress = false;
  }
}
```

付款完成後顯示：

```text
欠款已全部清償。
床鋪休息權限已恢復。
```

紅怡台詞可使用：

```text
「收到了。」
```

# 11. 付款交易完整性

付款必須避免以下錯誤：

* 快速連按造成重複扣款。
* 對話關閉時再次呼叫付款。
* 存檔中斷造成扣錢但欠款未清除。
* 欠款清除但玩家金錢未扣除。
* 場景切換造成同一筆付款執行兩次。

推薦處理方式：

1. 付款前建立交易鎖。
2. 再次驗證金錢與欠款。
3. 同一個函式內扣款並清除欠款。
4. 立即存檔。
5. 完成後解除交易鎖。

如果專案有統一 Transaction、Command 或 Save Queue 系統，應整合該系統。

---

# 12. 房租與主線任務關係

欠租不得阻止玩家：

* 與紅怡進行必要主線對話。
* 繳交主線任務物品。
* 取得 B1 通行權限。
* 取得或授權 B2 通行卡。
* 進入已合法解鎖的 B1 或 B2。
* 挑戰必要主線 Boss。
* 取得必要任務獎勵。
* 使用正式繳款入口。

欠租只直接限制：

```text
床鋪完整休息。
床鋪生命全回復。
床鋪主動跨日功能。
```

如果未來新增其他欠租懲罰，必須另外寫入規格，不得由實作者自行推測。

---

# 13. 每日世界刷新

休息成功或自然跨日後，可以刷新：

* 每日隨機事件
* 一般敵人配置
* 一般可重生物件
* 每日商店資料
* NPC 每日台詞
* 每日探索 Seed
* 非永久短期狀態

不得刷新或回退：

* 主線 Quest State
* 已取得的關鍵任務道具
* 已解鎖的 B1、B2 權限
* 已擊敗的主線 Boss
* 維修室永久完成狀態
* B1 淹水解決狀態
* B2 文件閱讀紀錄
* Demo Ending 進度
* 永久收集品

每日資料與永久資料必須分開處理。

---

# 14. 存檔規格

最低必要存檔欄位：

```ts
interface BedRentSaveData {
  day: number;
  timeOfDayMinutes: number;

  dailyRent: number;
  outstandingRentDebt: number;
  overdueDays: number;

  playerMoney: number;
  playerCurrentHealth: number;
  playerMaxHealth: number;
}
```

實際專案若已分散至不同 Save Module，不必建立單一物件，但資料行為必須一致。

不應保存：

```ts
isDayTransitioning
isRentPaymentInProgress
```

這兩項屬於執行期鎖定狀態。

載入存檔時應重設為：

```ts
isDayTransitioning = false;
isRentPaymentInProgress = false;
```

避免讀檔後永久無法休息或付款。

# 16. 禁止實作項目

Codex 不得自行加入以下功能：

* 部分付款
* 分期付款
* 自動扣款清償欠款
* 在床鋪付款
* 在 UI 遠端付款
* 其他 NPC 代收
* 房租道具抵扣
* 以任務完成自動清除欠款
* 欠租時免費睡眠
* 欠租時部分回血
* 欠租時付費回血
* 獨立保存 `bedLocked`
* 休息時重置主線進度
* 每次進入房間重複收租
* 床鋪確認畫面開啟時提前收租

除非後續正式規格明確增加，否則以上功能均不得實作。

# 20. 最終不可變規則

本系統的最終規則如下：

```text
1. 現實 12 分鐘等於遊戲內一天。
2. 玩家可在自己的床上主動結束當日。
3. 成功休息會回滿生命並進入次日房租結算。
4. 不論當日何時休息，皆直接前進至次日。
5. 只要存在任何欠租，床鋪完全禁止休息。
6. 欠租期間不能透過床恢復生命。
7. 玩家只能找紅怡繳交欠租。
8. 紅怡是唯一繳款入口。
9. 欠租只能一次全部付清。
10. 金額不足時不得扣除任何金錢。
11. 不接受部分付款或分期付款。
12. 欠款歸零後，床鋪休息權限立即恢復。
13. 床鋪權限必須直接由欠款是否為 0 推導。
14. 自然跨日與床鋪休息共用同一套每日結算。
15. 欠租不得阻擋必要主線內容與繳款入口。
```
