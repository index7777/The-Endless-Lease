# 《無期租寓》UI 美術最高規則 v1.0（禁止偏離）

最後更新：2026-07-16
制定者：製作人
效力：本文件是專案 UI Design System 的最高規範。凡 UI 美術、版面、元件、材質、色彩、字級、圖示、互動與動畫與其他文件衝突時，以本文件及其七張基準圖為準。

任何新頁面都必須讓玩家認為是同一位設計師、同一套元件庫完成。不得因功能不同更換 UI 風格，也不得為個別頁面重新發明框、按鈕、卡片或動畫。

## 1. 世界內實體文件原則

所有 UI 都不是抽象遊戲 HUD，而是管理室內具有實體存在感的物件：

- 登記表
- 檔案
- 契約
- 公文
- 建檔卡
- 管理紀錄
- 結算文件

所有資訊都必須像印刷品。禁止科技介面、發光框、現代玻璃、Material Design、Cyber、Neon，以及任何與管理室實體文件不相容的視覺語言。

## 2. 唯一材質系統

整個遊戲 UI 只允許三種材質：

1. 黑色金屬：外框、按鈕、抽屜、管理櫃。
2. 泛黃紙張：所有主要資訊、表單、卡片與公文。
3. 深色木頭：桌面、背景與管理室環境。

不得加入第四種材質。紙張必須有一致紋理、微弱厚度與接觸投影，不能是平面色塊，也不能像懸浮玻璃卡片。所有頁面使用同一套紙張紋理。

## 3. 固定版面結構

所有頁面必須遵守左／右結構：

- 左側：故事、身份、流程、說明、上下文。
- 右側：互動、數值、檔案卡、選項、確認動作。

禁止把所有內容集中在中央。每個區塊必須保留呼吸與留白；禁止把畫面塞滿或造成資訊爆炸。背景永遠使用真實場景，例如管理室、走廊、電梯；禁止純黑背景、模糊圖片與 Gradient。

## 4. 共用元件庫

所有新畫面只能由同一套共用元件組裝。最低正式元件如下：

- `PaperPanel`：泛黃紙張主面板，統一紙紋、厚度、磨損與陰影。
- `BlackFrame`：1–2 px 黑色金屬細框及必要角件。
- `ArchiveCard`：身份、天賦、缺陷、租房及其他檔案卡；尺寸、外框、紙張與內距一致。
- `ManagementButton`：像管理室公文抽屜／金屬按鍵；Hover 只略微變亮。
- `ProgressHeader`：行政流程導覽與目前步驟。
- `Stamp`：蓋章、已記錄、警告與核准狀態。
- `Divider`：細行政分隔線與文件段落結構。

身份、天賦、缺陷與租房卡必須使用相同卡片尺寸、外框、紙張與排版規則。不得突然出現不同卡片樣式。

## 5. 按鈕與互動

所有按鈕都是管理室實體按鈕或文件操作件，不是一般遊戲 Button。

- Hover：只允許略微變亮，表現管理員正在翻閱或準備操作。
- 禁止：放大、跳動、發光、Bounce、Elastic、Spring。
- 禁止使用 CSS `border-radius` 製造圓角。
- 若需要磨損、鐵框或紙角，必須來自統一素材與材質，而不是現代圓角卡片。

## 6. 圖示

所有 Icon 只能使用黑色、單色、線稿風格。

禁止彩色 Icon、3D Icon、Material Icon、Hero Icon，及與行政文件不一致的現代圖示套件外觀。

## 7. 字級

整個 Design System 只允許四種文字層級：

- `H1`：超大標題，例如「租約建檔」、「建檔完成」、「命運裁定」。
- `H2`：區段標題，例如「身份背景」、「天賦」、「缺陷」。
- `Body`：一般內容。
- `Caption`：小字、編號、註記與輔助說明。

不得新增第五種任意字級。所有頁面使用相同字體角色、字距、行距與層級比例。

## 8. 唯一色票

只允許以下色彩 token：

```text
Paper      #D8C8A8
Background #171513
Frame      #3C342D
Text       #1F1C1A
Warning    #7D2424
```

禁止藍色、綠色、紫色與亮黃色。亮暗變化只能是上述色票因材質、光線、磨損與互動狀態產生的合理變化，不得另立新主色。

## 9. 外框與陰影

- 所有外框統一為 1–2 px 細框。
- 禁止粗框、卡通外框與現代發光描邊。
- 紙張必須呈現微弱厚度、接觸陰影與真實承載關係。
- 紙張不可無依據漂浮；陰影方向與真實場景光源一致。

## 10. 動畫語言

所有 UI 動畫必須是行政動作：

- 紙張滑入
- 翻頁
- 印章落下
- 蓋章
- 建檔
- 文件收納與抽出

禁止縮放式轉場、Bounce、Elastic、Spring，以及像商城或手機 App 的動效。Hover 必須像管理室正在翻閱文件。

## 11. 電影畫面標準

每一頁都不是單純功能畫面或美術展示，而是電影畫面；任何狀態都應能直接截圖作為宣傳圖，同時保持功能清楚、文字可讀與操作可辨識。

## 12. 新頁面強制驗收

每新增或重製一個頁面，必須逐項比對七張基準圖：

- 紙張一致。
- 字體與四級字級一致。
- 陰影與實體接觸一致。
- 管理室按鈕一致。
- 真實場景背景一致。
- 1–2 px 黑色金屬外框一致。
- 黑色單色線稿 Icon 一致。
- 行政動畫一致。
- 左／右結構一致。
- 共用卡片尺寸與材質一致。
- 只使用五個色彩 token。
- 無 CSS 圓角、Gradient、發光框或第四種材質。

只要任一項不同，就必須重新製作；不得以「功能不同」作為偏離理由。

## 13. 實作與回歸限制

- 必須先抽取共用 Design Tokens 與上述元件，再組裝新頁面。
- 禁止在單一頁面另寫一套顏色、陰影、圓角、卡片或按鈕。
- 舊頁面若不符合本文件，標記為待重製，不得以既有程式為理由延續錯誤風格。
- UI 修改後須保留基準截圖，至少驗證 16:9、1366×768 與手機橫向；不得裁切、溢出、重疊或破壞左右結構。
- 「完成」必須包含視覺逐項檢查與實機畫面證據；只有建置或自動測試通過不算 UI 驗收完成。
