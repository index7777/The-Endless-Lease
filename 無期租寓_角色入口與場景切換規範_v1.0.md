 角色生成、房間入口與場景切換規範

## 1. 核心原則

角色進入任何房間時，初始位置必須由「具名入口點」與「地面偵測系統」共同決定。

禁止直接使用：

- 固定螢幕座標
- 背景圖片座標
- 房門圖像中心
- 上一個場景的角色座標
- 每個房間各自寫死的 X、Y 偏移
- 為了修正單一房間而加入的臨時特殊判斷

角色的生成位置必須代表角色腳底中心的世界座標，而不是角色圖片中心、頭部、碰撞體中心或房門中心。

任何需要針對單一房間手動修改角色高度的做法，都代表入口系統尚未完成。

---

## 2. 場景入口資料結構

每個可進入的房間都必須配置一個或多個具名入口點。

建議命名方式：

```text
Spawn_FromHallway
Spawn_FromRoom601
Spawn_FromElevator
Spawn_FromStairs
Spawn_Default

每個入口點至少必須包含：

spawn_id
world_position
facing_direction
ground_probe_distance
arrival_animation
camera_anchor
entry_transition
safe_radius

入口點的座標必須位於可行走區域內，並以角色腳底接觸點作為定位基準。

範例：

{
  "scene_id": "room_601",
  "spawn_points": [
    {
      "spawn_id": "from_hallway",
      "position": [176, 612],
      "facing": "right",
      "ground_probe_distance": 48,
      "arrival_animation": "door_enter",
      "camera_anchor": "room_center",
      "safe_radius": 24
    }
  ]
}

實際資料格式可依專案架構調整，但禁止把這些數值散落在不同程式檔內。

3. 出口與入口必須明確配對

每個房門、樓梯、電梯與場景出口，都必須明確記錄目的地及目的入口。

例如：

{
  "exit_id": "room601_to_hallway",
  "destination_scene": "floor6_hallway",
  "destination_spawn_id": "from_room601"
}

反方向：

{
  "exit_id": "hallway_to_room601",
  "destination_scene": "room_601",
  "destination_spawn_id": "from_hallway"
}

禁止只記錄目的場景，卻不記錄目的入口。

禁止進入場景後再用「找最近的門」或「使用第一個出生點」猜測角色位置。

4. 正確的角色生成流程

每次進入新場景時，Codex 必須依照以下順序執行：

鎖定玩家輸入。
開始淡出或門口轉場。
載入目的場景。
等待背景、碰撞、可行走區域與地面資料初始化完成。
根據 destination_spawn_id 找到正確入口點。
將 CharacterRoot 移動到入口點的世界座標。
執行向下地面偵測。
將角色腳底對齊實際地面接觸點。
驗證角色碰撞體沒有與牆壁、房門或家具重疊。
設定角色面向。
設定攝影機位置。
播放進門或落地動畫。
等攝影機與角色穩定後再解除玩家輸入。
完成淡入。

禁止在場景尚未完成碰撞初始化前放置角色。

禁止角色先出現在錯誤位置，再於下一幀瞬移至正確位置。

5. CharacterRoot 與 VisualRoot

角色必須使用以下分層：

CharacterRoot
├── CollisionShape
├── GroundProbe
├── VisualRoot
│   ├── CharacterSprite / AnimatedSprite
│   └── ContactShadow
└── InteractionArea
CharacterRoot

CharacterRoot 的世界位置代表角色腳底中心。

所有場景入口點只能設定 CharacterRoot。

VisualRoot

VisualRoot 只負責角色圖片、動畫及局部視覺偏移。

禁止用 VisualRoot 的位置作為存檔位置、入口位置或碰撞位置。

禁止為了讓角色看起來接近地板，而只移動圖片、不移動碰撞根節點。

6. 地面對齊

入口點只提供預期位置，最終位置必須由地面偵測確認。

角色生成後必須向下檢測：

ground_hit
ground_position
ground_normal
ground_distance
ground_collider

若成功偵測地面：

CharacterRoot.y = ground_position.y

實際計算需考慮引擎座標方向與碰撞體腳底偏移。

若沒有偵測到地面，禁止讓角色直接生成。

應依序：

嘗試入口點附近的小範圍安全搜尋。
嘗試入口點設定的 fallback。
使用 Spawn_Default。
記錄錯誤並顯示除錯資訊。

禁止無聲地把角色放在 (0, 0)、場景中心或畫面左上角。

7. 安全位置驗證

角色進入房間後，必須檢查：

腳底是否位於可行走地面。
碰撞體是否穿進牆壁。
是否和房門碰撞體重疊。
是否和床、櫃子、桌椅重疊。
是否生成在不可行走區域。
是否生成在遮罩後方或錯誤景深區域。
是否離入口過近，導致玩家立刻觸發返回。
是否位於攝影機可視範圍內。

若生成位置不安全，應在入口點附近搜尋最近的合法位置。

搜尋必須限制在合理半徑內，不可把角色移到房間另一側。

8. 房門防止重複觸發

角色剛進入房間時，不得立即重新觸發同一扇門。

必須採用以下至少一種方式：

角色離開門口觸發區後，才重新啟用出口。
進場後給出口短暫冷卻時間。
使用進出方向判定。
使用場景切換狀態鎖。

推薦條件：

exit_enabled =
transition_finished
AND player_has_left_entry_trigger
AND input_unlocked

禁止只用固定的 0.5 秒延遲處理所有情況。

9. 2D 背景場景的額外規則

若場景是預渲染背景或 2D 合成畫面，不能把背景圖片本身當成地面。

每個房間必須額外配置：

WalkableArea
GroundLine / GroundPolygon
CollisionBoundaries
OcclusionZones
DepthZones
InteractionZones

角色入口點必須位於 WalkableArea 內。

若角色會隨畫面深度改變縮放，入口點還必須包含或能查詢：

depth_value
visual_scale
sorting_layer

角色縮放只能影響 VisualRoot，不能任意縮放碰撞體或改變腳底世界位置。

目前畫面中角色出現在門邊半空，常見原因包括：

出生點設在門圖像中心。
角色 pivot 位於圖片中心。
場景使用螢幕座標而不是世界座標。
入口點建立在牆面而非地面。
場景載入後未重新執行地面對齊。
角色圖片與碰撞根節點有錯誤偏移。
背景縮放後，入口點沒有同步使用相同座標系統。

Codex 必須先確認根本原因，不得直接修改角色 Y 值碰運氣。

10. 攝影機與進場演出

角色進入房間時，攝影機應先定位至目的場景，再顯示角色。

推薦順序：

淡出
→ 載入房間
→ 放置角色
→ 對齊地面
→ 設定攝影機
→ 播放進門動畫
→ 淡入
→ 解鎖操作

禁止：

畫面淡入後角色才瞬移。
攝影機先從錯誤位置滑過來。
角色在空中播放 idle。
玩家可在地面對齊完成前移動。
角色生成時播放走路循環，但沒有實際位移。

進場動畫結束後，必須自然切換至 idle 或玩家輸入的移動動畫。

11. 存檔與讀檔位置

存檔時只能保存經過驗證的合法位置。

至少保存：

scene_id
last_valid_grounded_position
facing_direction
last_entry_spawn_id

禁止在以下狀態保存位置：

正在場景切換。
正在門口進出動畫。
角色尚未完成地面對齊。
角色位於碰撞重疊中。
角色正在被系統修正位置。
角色處於不可行走區域。

若讀檔位置失效，應回到該場景最近的安全入口點，而不是強制使用錯誤舊座標。

12. 必要除錯顯示

Codex 必須建立可開關的入口除錯模式，顯示：

所有 SpawnPoint 名稱。
SpawnPoint 世界座標。
SpawnPoint 面向方向。
CharacterRoot。
VisualRoot。
腳底 pivot。
碰撞體。
GroundProbe。
Ground hit point。
WalkableArea。
出口觸發區。
安全搜尋半徑。
目前使用的 destination spawn ID。
最終修正前後座標。
是否通過位置驗證。

建議除錯開關：

debug_spawn_points = true
debug_room_transitions = true
debug_ground_alignment = true

正式版本可關閉顯示，但除錯功能必須保留至遊戲開發完成。

13. 自動驗證

每個場景入口至少必須自動檢查：

入口 ID 是否唯一。
所有出口是否有有效目的場景。
所有出口是否有有效目的入口。
入口是否位於可行走區域。
入口下方是否能偵測到地面。
入口是否與牆壁或家具重疊。
角色生成後是否位於攝影機範圍內。
角色腳底與地面距離是否在容許值內。
角色是否會立即重新觸發出口。
不同角色尺寸是否仍可安全生成。
不同解析度與畫面比例下是否一致。

建議初始限制：

{
  "max_spawn_ground_gap_px": 1.0,
  "max_spawn_penetration_px": 0.5,
  "max_spawn_position_correction_px": 24,
  "entry_trigger_rearm_distance_px": 20,
  "max_camera_settle_time_seconds": 0.3
}

數值需依遊戲解析度與角色尺寸調整，並記錄調整原因。

14. 必測場景

角色入口系統至少必須通過：

601 房間由走廊進入。
601 房間離開至走廊。
管理室進出。
電梯進出。
樓梯上下樓。
一般住戶房間。
有門檻的房間。
出口旁有家具的房間。
不同角色動畫與角色尺寸。
存檔後重新進入。
死亡後重生。
場景快速連續切換。
30、60、120 FPS。
暫停後恢復。
視窗失焦後恢復。
15. 完成標準

只有同時符合以下條件，入口系統才可標記完成：

角色從每個入口進入時都站在正確地面。
角色不會出現在牆上、空中、床上或門框中。
角色腳底與碰撞體位置一致。
不需要為單一房間手動調整座標。
入口與出口具名配對。
地面偵測在場景初始化後執行。
角色不會在畫面中可見地瞬移。
角色不會立即重新觸發出口。
攝影機與進場動畫自然。
存檔與讀檔只使用合法位置。
除錯工具與自動檢查完整。
所有房間都使用同一套入口系統。

最終判定原則：

場景入口只決定角色應該從哪裡進來，地面與碰撞系統決定角色最後站在哪裡。不得把背景上的房門位置直接當成角色腳底位置，也不得靠每個房間的固定偏移修正錯誤。


再把下面這段當作 Codex 目前的修正任務：

```text
目前角色進入 601 房間後出現在左側門框附近的半空中，表示場景入口、角色 pivot、座標系統或地面對齊至少有一項設定錯誤。

請不要直接修改 601 房間的固定 Y 座標，也不要只移動角色圖片。

請依照 AGENTS.md 的「角色生成、房間入口與場景切換規範」進行完整排查：

1. 顯示 CharacterRoot、VisualRoot、碰撞體、腳底 pivot、SpawnPoint 與 GroundProbe。
2. 確認目前入口是否錯誤使用房門圖片中心或螢幕座標。
3. 確認 Spawn_FromHallway 位於 601 房間地面的可行走區域內。
4. 場景載入與碰撞初始化完成後，再放置角色並向下偵測地面。
5. 將 CharacterRoot 的腳底中心對齊 ground hit point。
6. 檢查角色碰撞體沒有與門框、牆壁或床重疊。
7. 設定正確面向與進門動畫。
8. 角色離開門口觸發區前，不得重新啟用返回走廊。
9. 將同一套系統套用到管理室、走廊、電梯與所有住戶房間。
10. 建立自動入口驗證，任何入口無地面、位置重疊或引用不存在的 spawn ID 時，建置或測試必須失敗。

完成後請提供：
- 問題根因。
- 修正前後座標。
- 使用的入口 ID。
- 地面偵測結果。
- 601 房間進出錄影或序列截圖。
- 管理室與走廊的交叉測試結果。

只有角色在所有入口都能穩定站在地面、沒有瞬移、沒有穿牆，而且不需要房間專屬補丁時，才能標記完成。
