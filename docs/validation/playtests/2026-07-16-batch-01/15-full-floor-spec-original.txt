《無期租寓》4F 走廊實機畫面修改規格
一、修改目標

目前畫面已具備完整場景，但仍有以下問題：

玩家角色與門、地面透視的關係不夠穩定。
玩家看起來像貼在背景前方，而不是站在走廊地面上。
拾取物名稱與角色重疊，閱讀混亂。
HUD 文字過小，尤其在 1920×1080 下不易辨識。
左右 HUD 與中央場景之間缺少統一比例。
房號文字像漂浮在畫面上，未完全貼合門牌。
畫面中下方存在過多空白，角色、物品與 UI 的焦點不明確。
場景雖然具有景深，但角色與物件的遮擋、陰影及縮放規則仍不像完整 2.5D 遊戲。

此次修改不得重做場景美術，只調整渲染、比例、錨點、HUD 與互動顯示。

二、角色與門的比例
1. 成人角色標準高度

以每扇房門的實際門洞高度作為統一比例基準。

普通成人角色高度 = 門洞高度的 76%–82%

本畫面角色建議落在：

約 79%

角色頭頂必須低於門框明顯距離，避免人物像與門同高。

禁止各場景手動填不同的角色圖片高度。

建立集中設定：

export const CHARACTER_SCALE = {
  player: 0.79,
  normalNpc: 0.78,
  tallNpc: 0.85,
  normalMonster: 0.8,
  rentCollector: 0.88,
  boss: 1.05,
};

這些數值應以門洞高度為相對基準，而不是直接使用固定像素。

2. 場景門高量測

為目前走廊背景建立明確的門洞資料：

type DoorReference = {
  room: string;
  centerX: number;
  groundY: number;
  topY: number;
  openingHeight: number;
};

不得只保存房號中心點。

角色比例驗收時，至少使用：

403 房門
405 房門
407 房門

確認三處比例一致。

三、角色貼地與腳底錨點
1. 統一腳底 Pivot

所有角色素材的 Pivot 必須位於：

雙腳與地面接觸區域的水平中心

不得使用圖片中心。

每一幀需保存相同的畫布尺寸與相同腳底錨點。

例如：

type SpriteFrame = {
  image: HTMLImageElement;
  footAnchorX: number;
  footAnchorY: number;
};

畫面位置計算必須是：

drawX = worldX - footAnchorX * renderScale;
drawY = groundY - footAnchorY * renderScale;

禁止用圖片高度直接猜測角色腳底。

2. 地面基準線

走廊需建立可行走地面區域：

const WALKABLE_GROUND = {
  nearY: ...,
  farY: ...,
  minX: ...,
  maxX: ...,
};

玩家腳底只能落在該區域。

角色移動時，位置值代表：

腳底位置

不是人物圖片左上角，也不是人物中心。

3. 接觸陰影

角色陰影不得包含在每一張 AI 動畫素材中。

統一由程式繪製橢圓接觸陰影：

位置：角色腳底下方
寬度：角色肩寬的 55%–70%
高度：寬度的 20%–28%
透明度：0.18–0.28
模糊：8–14 px

移動時陰影不能漂移。

角色跳躍、受擊或特殊離地時，才縮小並降低透明度。

目前畫面中角色腳下陰影偏像一塊獨立黑斑，需降低邊界清晰度並更靠近鞋底。

四、2.5D 景深縮放

目前背景具有透視，但角色尺寸看起來基本固定。

建立統一深度倍率：

function getDepthScale(groundY: number): number {
  return lerp(0.82, 1.08, normalizedDepth);
}

建議：

靠近走廊牆面／遠處：0.82–0.9
主要行走帶：0.94–1.02
畫面前景：1.03–1.08

限制：

玩家與普通怪物必須使用相同深度公式。
不可讓角色因動畫狀態改變尺寸。
不可在每個場景自行建立不同公式。
Boss 或追租者只能額外乘角色類型倍率。
五、角色動畫修正
1. 不以少量不同圖片硬切

每個角色動畫需有明確階段。

待機

至少 6–8 個有效姿勢或 12 幀播放序列：

吸氣
停留
吐氣
重心微移
回正

禁止只用整張圖片上下縮放模擬呼吸。

走路

至少包含完整八個關鍵姿勢：

左腳接觸
左腳下降
經過姿勢
左腳抬起
右腳接觸
右腳下降
經過姿勢
右腳抬起

最終播放可以補至 12–16 幀。

奔跑

至少 10–14 幀，需包含：

身體前傾
騰空
落地
袋子或衣物延遲運動
攻擊

至少包含：

預備
蓄力
出手
命中
過衝
恢復

建議總長：

0.42–0.58 秒

傷害只能在命中幀觸發一次。

受擊

不得只切換一張受傷圖。

需有：

命中反應
重心後移
短暫停頓
恢復
死亡

需要實際倒地階段，不能只淡出。

2. 幀間一致性

所有動畫幀必須通過以下檢查：

臉部不能改變身份。
服裝細節不能忽有忽無。
武器長度不能跳動。
包包位置不能每幀重新生成。
身體高度差不得超過 2%。
平均亮度差需控制。
色溫必須一致。
邊緣不能有不同程度白邊。
腳底錨點誤差不得超過 2 px。

建立動畫素材預處理程序：

去背
統一畫布
自動對齊骨盆
固定腳底
亮度標準化
色溫標準化
邊緣去白
輸出 Contact Sheet
六、拾取物與互動提示

目前角色腳邊同時出現：

藥品
其他

提示互相重疊，且遮住角色腿部。

修改規則
1. 平時不顯示物品名稱

地面物品未被選中時，只顯示物品素材與極淡輪廓，不顯示文字。

2. 只顯示最近一個可互動物

玩家進入互動距離後，只顯示距離最近且優先級最高的物品：

藥品

按 E 拾取

其他物品不可同時顯示名稱。

3. 提示位置

提示應放在物品上方：

距物品中心上方 32–48 px

不得貼在角色腳邊。

4. 顯示線或焦點框

可使用細線指向物品，但不能用大型黑框。

建議：

半透明深色底
1 px 暗金邊框
最多兩行文字
5. 物品重疊

同位置存在多個物品時：

藥品 ×2

按 E 拾取

或開啟小型選擇清單，不要將所有標籤堆在地面。

七、房號顯示

目前房號位於門上方，但部分房號看起來像 HUD 文字，而非場景門牌。

修改規則
房號必須綁定每扇門的 plaqueX、plaqueY。
房號不跟隨攝影機 HUD 層。
字體尺寸需依場景縮放。
房號顏色需受場景光照影響。
房號不可過亮。
407 暖燈門牌應略偏暖；其他門牌維持灰黃。
玩家靠近時才可額外顯示互動文字，房號本身保持場景化。

例如：

ctx.globalAlpha = 0.72;
ctx.fillStyle = door.lightTemperature === "warm"
  ? "#c8a56a"
  : "#b7aa87";
八、HUD 尺寸與可讀性

目前 HUD 整體過小，尤其：

左上本輪資訊
右上租約狀態
底部操作文字
快捷欄內容

在高解析度下不易讀取。

1. 使用統一 UI Scale
uiScale = clamp(
  viewportHeight / 900,
  0.9,
  1.25
);

不要讓 UI 僅依畫布原始尺寸固定縮放。

2. 最小字級

1920×1080：

一般 HUD 文字：不得低於 15 px
次要說明：不得低於 13 px
主要數值：20–30 px

1366×768：

一般 HUD 文字：不得低於 13 px
次要說明：不得低於 12 px

不得出現目前畫面中接近 9–11 px 的資訊文字。

3. 左上租金卡

整理為：

今日租金
19 租券

距離結算
08:21

既有欠款
0 租券

目前目標移到另一張小卡：

目前目標

前往 B1
清除底層異常源

不要把任務文字擠在租金卡同一區。

4. 右上租約狀態

目前字太小、面板太空。

調整為：

租約狀態

欠款　　　　　　0 租券
追租程序　　　　未啟動
承租房保護　　　有效
公寓關注度　　　0 / 10
監控狀態　　　　未留意

使用左右欄對齊。

5. 生命與體力

目前長條太靠畫面底部，數字過小。

改為：

生命　81 / 96
體力　102 / 102

數值和條形保持 4–6 px 間距。

畫面底邊至少保留 18–24 px 安全區。

6. 快捷欄

目前快捷欄位置尚可，但文字偏小且低對比。

提高：

名稱字級
數量清晰度
選中框亮度
禁用槽差異

操作提示：

WASD 移動　Shift 奔跑　Space 攻擊　E 互動

只在教學期間顯示，之後淡出。

九、畫面安全區

目前 HUD 太靠近四邊。

建立：

桌面安全區：24 px
1366×768：16 px
手機橫向：12 px

所有 HUD 不得貼齊畫面邊緣。

十、角色與場景色彩融合

角色目前比場景略銳利、偏亮，像獨立素材貼在背景上。

程式渲染角色時加入統一場景校色：

亮度：依區域光源調整 ±8%
飽和度：降低 5%–10%
對比：降低 3%–6%
局部色溫：依最近燈源調整

例如靠近 407 暖燈時：

角色右側增加微弱暖色。
角色陰影仍維持灰綠。
不要整張角色變橘色。

可建立簡單的燈源區域：

type SceneLight = {
  x: number;
  radius: number;
  temperature: "cold" | "warm";
  intensity: number;
};
十一、遮擋與渲染順序

所有場景物件與角色按腳底 Y 排序：

renderDepth = footY;

渲染順序：

背景牆面
背景門與固定家具
地面物品陰影
地面物品
角色接觸陰影
角色／怪物／NPC，依 footY 排序
前景遮擋物
世界空間提示
HUD

物品提示不能被角色身體遮住，但也不能永遠壓在所有 HUD 上方。

十二、Codex 執行流程

Codex 不得一次直接修改完成，必須依序：

量測門洞高度與地面線。
建立角色統一尺度表。
修正所有 Sprite 腳底 Pivot。
修正角色陰影。
加入深度縮放。
修正房號錨點。
修正物品提示選取規則。
放大並整理 HUD。
加入場景校色。
產生三種解析度截圖。
逐項驗收。

輸出：

outputs/validation/hallway-4f-1366x768.png

十四、量化驗收標準

完成後必須符合：

玩家高度為門洞高度的 76%–82%。
同一角色不同動畫幀高度差低於 2%。
腳底錨點誤差不超過 2 px。
角色停止時雙腳不滑動。
走路動畫不出現整體上下跳動。
角色陰影中心與腳底中心誤差不超過 4 px。
同時只顯示一個主要拾取提示。
物品提示不遮住角色腿部。
房號正確對應每扇門牌。
一般 HUD 文字在 1920×1080 不低於 15 px。
HUD 不貼齊螢幕邊緣。
角色顏色與場景燈光不產生明顯割裂。
切換待機、移動與攻擊時，不出現角色縮放跳動。
1920×1080、1366×768 與手機橫向均無文字溢出。
最優先修改項目

Codex 應先完成以下五項，不要先重做其他系統：

P0-1　建立門高與角色比例基準
P0-2　修正所有角色腳底 Pivot 與接觸陰影
P0-3　修正物品提示重疊
P0-4　放大 HUD 字級並統一安全區
P0-5　修正角色與場景的色彩融合
