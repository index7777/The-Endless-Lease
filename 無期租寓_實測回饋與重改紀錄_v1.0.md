# 《無期租寓》實測回饋與重改紀錄 v1.0

最後更新：2026-07-16
用途：集中保存製作人實機測試後要求重改、不得回退及仍待驗收的項目。新的 Codex 任務開始前必須先讀本文件。本文件放在專案根目錄並由 Git 追蹤，不得移回忽略版本的暫存輸出資料夾。

## 0. 紀錄規則

- 本文件只記錄「製作人已測試後提出的問題」、「明確要求重做／修正」及其驗證結果。
- 不得只在聊天、程式註解或其他規格尾端留下退改要求；收到新回饋後必須追加到本文件。
- 每項至少記錄：日期、來源／批次、重現情境、回饋摘要、修正要求、處理狀態、驗證證據。
- 狀態只使用：`待處理`、`處理中`、`待製作人實測`、`已由製作人驗收`、`不可回退`。
- 舊對話逐字內容若已無法取得，只能寫成「依既有交接文件重建」，不得冒充製作人原話。
