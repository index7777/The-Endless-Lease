# 附錄 III：《無期租寓》電影化演出與互動規範 v1.0

效力：最高。

本附錄規範所有：

- 劇情事件
- 管理室事件
- UI 演出
- 文件閱讀
- NPC 對話
- 特殊發現
- 異常事件
- 房間探索
- 玩家控制
- 鏡頭
- 音效
- 轉場

凡演出與其他文件衝突，以本附錄為準。

## 1. 核心原則

《無期租寓》不是傳統遊戲。

玩家不是觀看 UI。

玩家是在體驗：管理室正在對自己進行行政程序。

因此，所有資訊必須存在於世界，不是存在於 HUD、Popup 或 Menu。

玩家看到的是文件、公文、租約、住戶檔案、公告、管理紀錄，而不是一般遊戲介面。

## 2. 玩家控制權

玩家控制只有三種狀態：

### Gameplay

玩家完全控制。

### Administrative Event

管理程序接管。玩家不能移動，只能閱讀、翻頁與確認。鏡頭可以控制；不得開啟背包、攻擊或跳出事件。

### Cinematic

完全電影演出。玩家沒有任何控制，不得跳出，除非設定允許 Skip。

禁止一邊走路一邊 Popup；禁止玩家跑步時跳出通知；禁止邊戰鬥邊播放文件。

## 3. 所有事件必須發生於世界

禁止直接 Popup、黑底字幕、中央通知與 RPG 對話框。

事件必須有來源，例如：管理室文件、牆上公告、住戶手冊、桌面資料、紅怡遞出的公文、房門通知、電梯公告、租約。

## 4. 文件演出

所有文件不得直接出現，流程固定：

玩家失去控制
→ 鏡頭找到文件來源
→ 文件被拿出
→ 放置桌面
→ 鏡頭推近
→ 紙張展開
→ 閱讀
→ 翻頁（若有）
→ 確認
→ 文件收回
→ 玩家恢復控制

禁止紙張 Fade In、紙張 Zoom 或紙張從天空掉下。

## 5. 管理室事件

所有重要資訊必須回到管理室，例如：新租約、特殊樓層、身份建檔、權限、通知、檔案與特殊權限。

管理室就是行政中心。禁止玩家在走廊收到正式公文。

## 6. 紅怡演出

紅怡永遠冷靜、優雅，永遠沒有多餘動作。

她不會大叫、揮手、劇烈表情、大幅轉身或快速走動。

所有動作都像真正管理員，例如：翻文件、蓋章、抽出資料、拿鑰匙、整理桌面、看向玩家、停頓、微笑。

所有演出必須留白。她不會說很多：一句，停；一句，停。

## 7. 對話規則

禁止一次出現大量文字。

每一句最多兩行。說完後由玩家確認，再出現下一句。

若需要大量資訊，改成文件，不要改成對話。

## 8. 鏡頭

禁止快速切鏡、劇烈 Zoom、搖晃與 FPS Horror。

鏡頭必須像電影：慢、穩、克制。

- 管理室：固定鏡頭。
- 文件：慢推。
- 公告：微推。
- 蓋章：近景。
- 回控制：慢退。

## 9. 音效

沒有音樂主導，只有世界，例如：紙張、腳步、翻頁、印章、燈、電梯、電話、鑰匙、管理室。

玩家永遠先聽見世界，不是聽見 UI。

## 10. UI 出現規則

所有 UI 必須有理由，例如：

- 背包：玩家翻住戶包。
- 住戶紀錄：玩家打開檔案。
- 寄存：管理室抽屜。
- 公告：公告板。
- 結算：正式結案報告。

這些不是一般 UI。

## 11. 異常事件

異常不是 Jump Scare，而是行政程序開始出錯，例如：文件內容改變、房號增加、日期消失、簽名不同、管理章變紅、通知自己翻頁、管理室變安靜、燈光改變。

玩家永遠先懷疑，再確認，最後恐懼。

## 12. 演出節奏

禁止連續事件。每個事件必須留呼吸。

例如：玩家走 → 停 → 管理室 → 文件 → 閱讀 → 停 → 恢復 → 再探索。

節奏必須慢、克制、壓迫。

## 13. 玩家恢復控制

所有事件結束前必須依序：

文件收起
→ 鏡頭退回
→ 玩家看到角色
→ HUD 恢復
→ 控制恢復

禁止閱讀完立即開始走路。

## 14. 世界一致性

所有新增內容，Codex 必須先確認是否符合 UI 美術最高規則、商業品質規範與電影化演出規範。

若任一項衝突，不得新增、不得自行設計，必須沿用既有 Design System。

## 15. Codex 不得自行設計新 UI（最後一條最高原則）

七張正式基準圖為唯一 UI 真值。

所有新畫面必須拆解既有元件（PaperPanel、ArchiveCard、ManagementButton、BlackFrame、Stamp、ProgressHeader、Divider）重新組裝，不得新增新的按鈕樣式、紙張樣式、卡片樣式、Dialog 樣式、通知樣式或動畫語言。

若新需求無法使用既有元件完成，Codex 應停止實作並提出設計衝突，而不是自行創造新的視覺風格。

任何畫面完成後，都必須與七張正式基準圖逐項比對，達到「玩家無法辨認這是新做的頁面」才可視為驗收通過。
