# 《無期租寓》固定視覺回歸矩陣

每次送驗固定使用 `1920×1080`、`1366×768`、`844×390（手機橫向）`。輸出放在 `outputs/validation/`，任何差異只可標示 `NEEDS_REVIEW`，不得自行視為製作人通過。

## 自動流程

執行 `npm run visual:capture`。需先提供：

- `PLAYWRIGHT_PACKAGE_PATH`：Playwright 套件位置。
- `VALIDATION_BASE_URL`：待測本機或部署網址。

目前腳本固定擷取：首頁、入住登記、命運裁定、選擇建檔、身份確認、建檔完成、初始承租房、置物櫃、住戶檔案／背包、暫停文件、存檔簿。

## 必測世界場景

以下場景每次玩法、碰撞、動畫或 UI 變更後都必須人工補齊三解析度證據：

1. 一般樓層公共走廊：門高、角色 Ground Line、左右邊界、電梯入口。
2. 承租房：床／角色比例、左側門碰撞、個人儲物櫃。
3. 住戶診療台：事件來源、診療台碰撞、互動熱區。
4. 電梯：轎廂移動邊界、控制盤互動、世界內 LED、管理室／10F 按鈕。
5. 管理室：獨立電梯按鈕、紅怡、專屬背景、文件與字幕。
6. B1：右側電梯出生、維修空間路由、Boss／玩家比例、Boss Collapse。
7. B2：右側電梯出生、Boss 掉落、權限卡、可回訪、最終流程。
8. 戰鬥：玩家 Walk／Attack、五類敵人 Walk／Attack／Hit／Die、Foot Lock、亮度差。
9. 拒絕交租：世界內逾期通知與跨場景追租者。
10. Demo 終局：查看紀錄後仍停留終局，不能回到新入住關卡。

## 量測門檻

- 成人角色：256 px；標準門：312 px。
- 腳底 Ground Line 誤差：不超過 2 px。
- B1 Boss：本 Demo 非巨型角色，渲染高度不得高於玩家。
- UI 安全區：所有可操作按鈕在三解析度內可見且可點。
- 動畫：24 FPS 播放；任何候選素材仍須通過 Animation Validator 與製作人實機驗收。
