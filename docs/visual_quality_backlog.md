# 視覺品質阻擋項目

目前首頁與入住登記屬功能驗證版，不得標記為美術完成。

## 已確認問題

- 首頁背景、Logo、管理室招牌、通知單與按鈕的材質細節不在同一品質層級。
- 四字 Logo 比例過大、亮度過高，與場景景深及角色構圖互相搶焦點。
- 小型 CSS 邊框和純色區塊破壞老舊紙張、金屬及印刷物的實體感。
- 入住通知單背景已含文字與框線，HTML 又重疊第二套文字和表單，造成殘影、歪斜、裁切與基準線錯位。
- 固定文案、動態欄位與互動熱區沒有共用同一套座標規格。
- 中文字級、行距、字距、按鈕標題位置未嚴格沿用參考圖。
- 六骰目前是屬性卡內的簡化線稿模型；應改為獨立共享場景中的六顆 3D 骰子投擲，落定後才更新純數值結果卡。
- 身份確認與建檔完成畫面沿用重複拉伸的簡化紙卡，未達參考圖的大型檔案、封印、章記與材質層級。
- 玩家、敵人與 NPC 目前只做少量姿勢切換，缺少正式走路、攻擊、受擊與死亡的完整主要幀及相鄰幀。
- 攻擊時的程式繪製弧線與寫實美術不一致，必須移除並由動作動畫本身表現揮擊。

## 重製規則

- 逐張參考圖建立 16:9 固定構圖與座標標註，不以概略 CSS 排版替代。
- 場景、紙張、金屬、污漬、壓印與固定文案合成為完成素材；所有中文字逐字校對，不接受生成亂碼。
- HTML 只保留玩家姓名、性別狀態、數值及可操作熱區等動態內容。
- 動態元素必須依素材規格使用固定 anchor、baseline、寬高與安全區，不得重新置中或自由漂移。
- 每個畫面需以參考圖並排對照，檢查構圖、色溫、對比、文字對齊、材質與視覺層級後才能通過。
- 角色動畫需以固定 CharacterRoot、同一骨架／母版、固定 baseline 與 pivot 產生，並通過腳底、輪廓、武器長度及循環抖動驗證。
