# 《無期租寓》2.5D Animation Bible（最高規範）v1.0

最後更新：2026-07-16
規則層級：所有角色動畫的最高規範，高於任何 AI 生成素材、舊動畫規格、既有程式與舊完成紀錄。
適用範圍：玩家、NPC、紅怡、普通怪物、追租者與所有 Boss。

任何素材不符合本文件時，必須標記 `Rejected` 並重新生成或重製，不得直接接入正式遊戲。

## 1. AI 圖片只是 Animation Source

AI 圖片只能作為 Base Frame／Animation Source，不能直接播放或視為最終動畫。AI 只負責繪製來源；Codex 必須負責比例、對齊、Ground、Pivot、Frame、Interpolation、光影、陰影、碰撞與遊戲內驗收。

禁止一次生成整套 Walk 後直接播放。若 AI 提供 16 張來源幀，必須重新對齊、品質篩選、插幀至目標幀數，再建立正式 Animation Clip。

## 2. Character Scale Reference

所有人物比例固定，不得依背景圖片任意自動縮放。

```text
成人參考身高：180 cm
成人遊戲高度：256 px
標準門高：220 cm
標準門遊戲高度：312 px
成人頭頂與門框淨空：15–25 cm（約 21–36 px）
```

任何人物放到標準門旁，頭頂都必須低於門框約 15–25 cm。超過門框或比例不符時必須重新縮放。

## 3. Ground Line、Pivot、Anchor 與 Foot Lock

- 每個場景建立 Ground Line。
- 所有角色腳底永遠貼住 Ground，禁止浮空、陷入地板或因動畫改變 Ground。
- 站立時左右腳必須 Foot Lock；呼吸不能用整張人物圖上下縮放或漂移。
- 所有動畫使用固定 Anchor；Hip、肩膀、眼睛、腳掌須跨幀 Auto Align。
- Sprite Pivot 必須位於兩腳之間的腳底中心，不得使用圖片中心。
- 地面高度改變時必須用 IK／Ground Probe 重新貼地。

## 4. 動畫最低幀數與動作內容

### Idle

- 玩家 Idle：至少 12–16 Frames。
- NPC Idle：至少 16 Frames。
- 內容必須是局部呼吸、肩膀、頭部、眼睛、衣服、手與重心微動。
- 禁止用整張圖縮放冒充 Idle。

### Walk

- 至少 16 Frames。
- 必須包含左腳接觸、重心轉移、右腳接觸、重心轉移。
- 禁止只交換左右腳或播放 8 Frames 循環。

### Run

- 至少 20 Frames。
- 必須包含 Body Lean、Bag Swing 與衣物延遲。

### Attack

- 至少包含 `Prepare → Swing → Impact → Recovery` 四個階段。
- 禁止三張圖或直接跳到砍擊姿勢。

### Hit

- 身體必須向後並產生合理重心偏移。
- 禁止只切換一張受傷圖。

### Die

- 必須是腿、腰、肩、頭依序 Collapse，最後停止。
- 禁止以 Fade 代替死亡動畫。

## 5. 統一 Animation State

所有角色使用同一套狀態命名與接口：

```text
Idle
Walk
Run
Attack
Hit
Die
Interact
Turn
Sit
OpenDoor
CloseDoor
```

不得讓每個角色各自發明不同狀態。尚未需要的狀態可以標記未製作，但資料接口必須存在。

## 6. 角色特有節奏

- 紅怡：所有動作速度固定為一般 NPC 的 50%，保持緩慢、行政化與完全從容。
- 滲牆住戶等怪物：不得使用人類等速步態；移動必須具有 `Move → Pause → Move → Pause` 的不適停頓。
- 追租者：禁止跑步；永遠維持走路，但實際移動速度比玩家快 5%。
- 所有怪物動畫至少 24 FPS，不得以 8 FPS 播放。

## 7. Frame Normalize 與 Auto Align

AI 來源幀不得直接播放。每張來源都必須經過：

- Brightness Normalize
- Contrast Normalize
- White Balance Normalize
- Shadow Normalize
- 眼睛 Alignment
- 肩膀 Alignment
- Hip Alignment
- 腳掌 Alignment
- 統一人物高度與輪廓比例
- Optical Flow 或 Frame Interpolation 補幀

AI 幀出現腳漂、人物忽胖忽瘦、光影不同、比例不同、門比例混入、肢體變形或閃爍時，該幀必須 Reject。

## 8. 陰影

禁止使用 AI 每張圖片自帶且不一致的陰影。人物陰影由遊戲系統統一建立，例如固定 Ellipse Shadow：

```text
形狀：Ellipse
Opacity：25%
Blur：由場景 Lighting Profile 控制
位置：CharacterRoot 腳底中心
```

陰影必須與 Ground、Pivot 及場景光線一致。

## 9. 固定製作流程

```text
AI 生成角色來源
        ↓
比例校正（Character Scale Reference）
        ↓
Pivot 與腳底 Ground Line 對齊
        ↓
統一光影、色溫、亮度
        ↓
Frame Alignment（骨盆、肩膀、腳掌）
        ↓
補幀（Optical Flow／Frame Interpolation）
        ↓
建立完整 Animation Clip
        ↓
檢查 Root Motion、Foot Lock、碰撞盒
        ↓
接入 Animator State Machine
        ↓
遊戲內驗收
```

任何階段都不得跳過。

## 10. Animation Validator

每次新增或修改角色時，必須執行 `npm run validate:animation`。正式候選素材必須再通過 `npm run validate:animation:strict`。

Validator 至少自動檢查：

- 人物高度是否符合 Character Scale Reference。
- 腳底是否始終貼住 Ground。
- 每幀 Pivot／中心點是否漂移。
- 尺寸、輪廓與人物比例是否突變。
- 光影、亮度與色溫是否一致。
- 動畫幀數與 FPS 是否達標。
- 必要狀態與 Attack 四階段是否存在。
- 是否出現跳幀、縮放、變形或閃爍風險。
- Root Motion、Foot Lock、碰撞盒與遊戲陰影資料是否完整。

只要一項不符合，結果就是 `Rejected`。未註冊 Manifest 的舊 `sprite-*.png` 一律是 `Rejected／Prototype Only`，不能因目前 Demo 仍引用就視為正式素材。

## 11. 完成標準

只有同時具備來源、逐幀輸出、規格 Manifest、固定比例、Ground、Pivot、Anchor、光影正規化資料、完整 Clip、Animator State、碰撞盒、Validator `Accepted` 報告與遊戲內實測證據，才能標記完成。
